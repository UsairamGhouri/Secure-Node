1. Our Commitment

The Secure Node team takes security seriously. We value the contributions of security researchers and the broader community in helping us maintain the safety and integrity of our platform. This policy outlines how to responsibly disclose security vulnerabilities.

2. How to Report a Vulnerability

If you discover a security vulnerability, please report it responsibly:

Report vulnerabilities to: mr.knight4837@gmail.com

When reporting, please include:

  • A detailed description of the vulnerability
  • Steps to reproduce the issue
  • Potential impact assessment
  • Any proof-of-concept code (if applicable)
  • Your contact information for follow-up

3. Scope

In Scope

  • Authentication bypass or privilege escalation
  • Cross-Site Scripting (XSS)
  • SQL Injection
  • Cross-Site Request Forgery (CSRF) bypass
  • Session hijacking or fixation
  • Insecure cryptographic implementations
  • Information disclosure of sensitive data
  • Remote code execution
  • Server-Side Request Forgery (SSRF)

Out of Scope

  • Denial of service attacks (rate limiting is already implemented)
  • Social engineering attacks
  • Physical security vulnerabilities
  • Issues in third-party dependencies with no practical exploit
  • Missing security headers on non-sensitive pages

4. Severity Classification

Severity Description Response Time
CRITICAL RCE, auth bypass, data breach 24 hours
HIGH Privilege escalation, XSS, CSRF bypass 48 hours
MEDIUM Information disclosure, session issues 5 business days
LOW Minor info leaks, best practice violations 10 business days

5. Safe Harbor

🛡 Safe Harbor Statement

We consider security research conducted in accordance with this policy to be authorized, lawful, helpful, and welcome. We will not initiate legal action against researchers who:

  • Make a good faith effort to avoid privacy violations, data destruction, and service disruption
  • Only interact with accounts they own or with explicit permission
  • Do not exploit vulnerabilities beyond what is necessary for the report
  • Report vulnerabilities promptly and do not publicly disclose before remediation
  • Do not use automated scanning tools that generate excessive traffic

6. Disclosure Timeline

  • Day 0: Vulnerability reported
  • Day 1-2: Acknowledgment sent to researcher
  • Day 3-7: Vulnerability assessed and triaged
  • Day 7-30: Fix developed and tested
  • Day 30-90: Public disclosure (coordinated with researcher)

7. Recognition

We believe in recognizing the contributions of security researchers. With your permission, we will:

  • Credit you in our security advisories
  • Add your name to our Security Hall of Fame (if applicable)
  • Provide a reference letter for academic or professional purposes
